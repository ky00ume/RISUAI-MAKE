# 2. CSS·HTML 위치 및 현재 문제

## 현재 발생한 문제 (이미지 기준)

CSS 텍스트가 채팅창에 그대로 출력되고 있음.

**원인**: 제가 전달한 CSS 코드가 중간에 잘려서 불완전하게 제공됨.
마지막 줄이 다음과 같이 끊김:

```css
.glass-card--char .glass-card__portrait {
  border-color:
```

`border-color:` 값이 없어 CSS 파싱 오류 발생.
이후 전체가 텍스트로 출력된 것.

## 해결책

백그라운드 임베딩의 내용을 **전부 지우고** 완성된 CSS로 교체.
(섹션 3에 완성본 CSS 수록 — 현재 미완성, HTML/CSS 작업 재개 시 완성 예정)

## CSS 작성 규칙

```
❌ 틀린 방법 — <style> 태그 포함해서 넣기
<style>
.glass-card { ... }
</style>

✅ 올바른 방법 — CSS만 그냥 붙여넣기
.glass-card { ... }
```

## HTML 작성 규칙

```
❌ 틀린 방법 — 캐릭터 카드 안에 넣기
❌ 틀린 방법 — 퍼스트 메시지에 넣기 (타이틀 제외)

✅ 올바른 방법 — 디스플레이 수정(editDisplay) 정규식에 넣기
  IN: ^(?=[\s\S]*\[META\])   (META 있는 메시지에만)
  OUT: [glass-card 전체 HTML]
  타입: 디스플레이 수정
```

# 3. 스마트폰 UI 구현 방향

## 구조 확정

```
[glass-card 레이아웃]
┌─────────────────────────────────────────────────┐
│  [glass-card__left]    │  [glass-card__right]   │
│                        │                        │
│  [캐릭터 프사]          │  [risutextbox]         │
│  [캐릭터 이름]          │                        │
│                        │  서사 텍스트...         │
│  ┌──────────────────┐  │                        │
│  │  스마트폰 UI      │  │                        │
│  │  (항상 표시)      │  │                        │
│  │                  │  │                        │
│  │  [탭1][탭2][탭3] │  │                        │
│  │  [탭4][탭5]      │  │                        │
│  └──────────────────┘  │                        │
└─────────────────────────────────────────────────┘
```

## 탭 구성 (5탭)

| 탭 | 아이콘 | 내용 |
|----|--------|------|
| status | 📊 | 배터리·신호·시간·층·위치·동행·호감도 |
| inv | 🎒 | 소지품 + 단서 목록 |
| contacts | 📞 | 5인 연락처 |
| msg | 💬 | 단톡방 + 개인톡 |
| gallery | 🖼️ | CG 슬롯(텍스트 대체) + 업적 |

## 탭 전환 방식

**CSS `:checked` 트릭** — JS 없이 순수 HTML/CSS로 탭 전환

```html
<!-- 숨김 라디오 버튼 -->
<input type="radio" name="x-risu-phone-tab" id="x-risu-tab-status" checked>
<input type="radio" name="x-risu-phone-tab" id="x-risu-tab-inv">

<!-- 탭 패널 -->
<div class="x-risu-tab-panel x-risu-tab-panel--status">...</div>
<div class="x-risu-tab-panel x-risu-tab-panel--inv">...</div>

<!-- CSS -->
#x-risu-tab-status:checked ~ ... .x-risu-tab-panel--status { display: flex; }
```

**제약 사항**:
- 탭 상태가 Lua 변수에 저장되지 않음
- 폰 닫았다 열면 항상 첫 탭(status)으로 초기화됨
- **허용하기로 결정함**

## CBS 주의사항

HTML 안의 CBS 변수는 RisuAI가 자동으로 처리함.
단, class 속성의 값이 `x-risu-` 접두사로 자동 변환되므로
CSS 선택자도 `.x-risu-클래스명`으로 작성해야 함.

```css
/* ❌ 이렇게 쓰면 안 됨 */
.glass-card { ... }

/* ✅ HTML에서 class="glass-card"를 쓰면
   RisuAI가 자동으로 x-risu-glass-card로 변환하므로
   CSS도 이렇게 써야 함 */
.x-risu-glass-card { ... }

/* 단, glass-card는 기존 글래스모피즘 테마에서 이미
   x-risu- 없이 사용 중이므로 기존 코드 그대로 유지 */
```

## 호감도 별표 출력 방식

CBS로 0~5 별표 출력:

```html
{{#when::{{getvar::cv_aff_oh}}::>=::1}}★{{/when}}
{{#when::{{getvar::cv_aff_oh}}::>=::2}}★{{/when}}
{{#when::{{getvar::cv_aff_oh}}::>=::3}}★{{/when}}
{{#when::{{getvar::cv_aff_oh}}::>=::4}}★{{/when}}
{{#when::{{getvar::cv_aff_oh}}::>=::5}}★{{/when}}
```

### Act 1 — 집결 (step 1~2)

**배경**: 5F 게스트 라운지. 비상등만 켜진 공간. 낯선 얼굴들.

**step 1 — 라운지 집결**
- 모든 NPC 외형으로만 묘사.
- 셔츠 차림 남자(서준혁)가 상황 설명 주도.
- 플레이어의 첫 응답 이후 자기소개 제안.
- 분위기 장치: 간헐적 진동, 먼지, 붉은 비상등, 먼 사이렌.

**step 2 — 자기소개 / 번호 교환**
- 순서: 서준혁 → 이나경 → 강태오 → 정나비 → 오하은 → 플레이어
- 번호 교환 완료 → 단톡방 **"더 헤리티지 한양"** 자동 생성 → `intro_done`

**분기 씨앗 (step 2에서 심기)**
- 정나비: 카메라 언급. 무릎 부상. 말이 많다.
- 오하은: 이름만 말하고 말을 아낀다. 창가 위치.
- 강태오: 투자자 신분. 아재개그. 플레이어를 슬쩍 관찰.
- 이나경: 프런트 직원. 구급상자 꺼내며 정나비 처치.
- 서준혁: 비상계단으로 마지막 합류. 먼지 묻은 소매.

### Act 2 — 자유탐색 (step 3~4)

**배경**: 5F 라운지를 거점으로 4F~8F 자유 이동.

**step 3 — 탐색 시작**

상층부 유도 장치:
- 서준혁: "위층이 구조 가능성 높습니다."
- 강태오: "아래는 끝났어. 위로 가자."
- 물리적: 4F 연기, 3F 이하 잔해 차단.

**도구 + 행동 요구 단서 시스템**:

| 도구 | 획득 방법 | 활용 |
|------|----------|------|
| 소방 도끼 | 4F 소방함 유리 파손 | 5F 514호 뒤틀린 문틀 강제 개방 |
| 손전등 | 5F 라운지 비상함 | 6F 수영장 정전 탐색, B2 진입 |
| 마스터키 | 이나경 호감도 3+ | 1F 지배인실, 7F 직원 구역 |
| 라이터 | 강태오 소지품 (호감도 2+에서 빌림) | 2F 벽지 뜯기 (균열 확인) |
| 렌치 | 6F 기계실 작업대 | 배관 차단, B1 밸브 조작 |

단서 발견 원칙:
- 단서는 행동 없이 자동 발견되지 않음.
- 도구가 있어야만 열리는 공간·단서가 존재.
- 한 응답에서 단서 발견 최대 1개.

**step 4 — 단서 축적 (단서 2개 이상)**
- 단톡방 "더 헤리티지 한양" 활성화.
- 개인톡 해금 시작 (호감도 2+).
- 이나경 개인톡: "이상한 거 봤는데 혹시 시간 돼요?"
- 정나비 개인톡: "사진 보고 있었는데 좀 이상한 거 아니에요?"
- 오하은: 6F 기계실 방향을 유독 서성인다.
- 서준혁: 비리 관련 장소 근처에서 화제를 돌린다.

**분기 포인트 A: 6F 수영장 탐색**
- 손전등 없이 진입: 배드엔딩 `pool_dark` 위험
- 손전등 있으면 탐색 가능
- 물+배선 구역 경고 무시: 배드엔딩 `electrocution`/`drowning` 위험
- 오하은 동행 시: 기계실 서류함에 먼저 손을 뻗는다

**분기 포인트 B: 강태오 의심도**

의심도 상승 조건 (확정):
- 아직 발견하지 않은 단서를 이미 아는 척 발언
- 1회차에서 B1·B2층 존재 언급
- 서준혁 비리를 단서 없이 단정 발언
- 아직 밝혀지지 않은 피해자(이상철) 정보 선제 언급
- **메타 발언: "저번엔 여기 열려있었는데 왜 닫혀있지?" 같은 다회차 지식 노출**
- **비밀번호/잠긴 문을 단서 없이 바로 정확히 시도**
- 단톡방에서 B2층/원본도면/금고실 언급: +2

의심도별 강태오 반응:
- 0: 무경계. 털털. "어, 그래?"
- 1: 한 박자 늦게 반응. "...그래?"
- 2: 농담이 줄어든다. "근데 그건 어디서 들었어?"
- 3: 표정이 굳어있다. "하나만 물어볼게."
- 4: 노골적 적대. "야, 이 호텔 알고 왔지?"
- 5: 확신. → 배드엔딩 `accomplice`

### Act 4 — 클라이맥스 (step 6)

**배경**: Phase 3 돌입. 전원 8F 루프탑 이동 직전.

**핵심 씬: 오하은의 살의 행동화**

- **막으면**: 전원 같이 8F 올라가서 구조 성공. 검거 씬 없음.
- **못 막으면 (1회차)**: 서준혁 살해 → 노말 루트로 진입.

플레이어 개입 선택지:
- 호감도 4+이고 2회차: 저지 가능 → `ach: blade` → 진엔딩 루트
- 호감도 3 이하이고 개입 시도: 배드엔딩 `betrayal` 또는 `silenced`
- 1회차: 어떤 선택도 저지 불가 (구조적으로 막힘)

### Act 5 — 엔딩 분기 (step 7~9)

**노말 A (normal_a)** — "검거"
- 조건: 단서 4개 이상 보유
- 씬: 강태오가 오하은에게 수갑을 채운다. 서준혁은 난간 너머 혼자 서 있다. 비리는 묻힌다. 헬기가 온다.
- 이후 이야기:
  - 오하은: 구치소. 첫 면회는 오지 않는다.
  - 강태오: 보고서를 제출했다. 3주 후 묻혔다.
  - 서준혁: 아무도 모르는 곳에서 술을 마신다.
  - 이나경: 퇴직서를 냈다.
  - 정나비: 그 사진을 편집하지 않는다.

**노말 B (normal_b)** — "단순 탈출"
- 조건: 단서 3개 이하
- 씬: 헬기가 왔다. 살아남았다. 아무것도 해결되지 않았다. 건물은 무너진다.
- 이후 이야기:
  - 모두: 살았다.
  - 진실: 땅과 함께 무너진다.
  - 서준혁: 재건축 허가가 났다는 뉴스가 나온다.

## 2회차 — 진실의 인양

> 1회차 노말엔딩 달성 후 이어하기. `cv_round = 2`.

**분기 포인트 F: 오하은의 살의 수면화 (step 5)**

**오하은 살인 저지 루트**

  - 호감도 3 이하: 배드엔딩 `silenced`.

### Act 4' — 최후의 선택 (step 8~9)

**진엔딩 조건 최종 체크 (5가지 모두 충족)**:

이후 이야기:

├── 노말 C (조건 부분 달성) ─────────── 서준혁 자수, 비리 폭로
└── 배드엔딩 30종 ────────────────────── 각 조건별
