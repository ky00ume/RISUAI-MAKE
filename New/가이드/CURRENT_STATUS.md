## 파일 현황

> 실제 프로젝트 파일들의 트리와 각 파일의 한줄 설명을 기록합니다.

```
프로젝트명/
├── 가이드/              (가이드 문서)
├── lua/                 (Lua 스크립트)
│   └── 01_main.lua      설명
├── css/                 (스타일시트)
│   └── 01_style.css     설명
├── regex/               (정규식 스크립트)
│   └── 001_panel.json   설명
├── html/                (퍼스트 메시지)
├── lorebook/            (로어북 데이터)
├── globalnote/          (글로벌노트)
├── risu/                (빌드 출력)
├── _tools/              (유틸리티)
├── Image/               (에셋 이미지)
├── build.ps1            빌드 스크립트
└── CLAUDE.md            핸드오프 문서
```

## 주요 데이터 흐름

> 시스템의 핵심 플로우를 텍스트 다이어그램으로 기록합니다.

```
[유저 입력]
  → onInput (Lua) → 전처리
  → onStart (Lua) → 프롬프트 수정
  → AI 응답 생성
  → onOutput (Lua) → 태그 파싱, 변수 처리
  → editDisplay (정규식) → UI 렌더링
  → 화면 출력
```
