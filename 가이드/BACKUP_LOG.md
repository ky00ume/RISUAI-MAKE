# 백업 로그

> 수정 전 백업 이력을 관리하는 문서입니다. 모든 파일 수정 전 반드시 백업을 생성하세요.

---

## 백업 규칙

### 넘버링
- 3자리 숫자: `001_`, `002_`, `003_` ...
- 같은 번호 = 같은 시점의 백업

### 백업 위치
- **파일별 백업**: 작업 파일은 해당 폴더의 `백업` 서브폴더에 저장
  - 예: `lua/01_main.lua` → `lua/백업/001_01_main.lua`
  - 예: `css/01_style.css` → `css/백업/001_01_style.css`
  - 예: `regex/001_panel.json` → `regex/백업/001_001_panel.json`

### 백업 타이밍
- 파일 수정 **직전**에 현재 상태 백업
- 대규모 작업 시작 전 관련 파일 일괄 백업
- 배포 전 전체 백업

---

- **lua/백업/000_**: (초기 파일 목록)
- **css/백업/000_**: (초기 파일 목록)
